Connect Case Study

Challenge

To develop a progressive web application with React and the Google Calendar API using a test-driven development technique. The app will be able to search for upcoming events by city, and view event information. Cached information will be able to be viewed offline!

Action

I created user stories, authentication, and various tests. I also visualized data, and styled the application.

Result

A stronger understanding of serverless architecture, progressive web apps, and test-driven development.

MyProcess

  1. Created user stories using the "Given-When-Then" syntax based on the applications intended features
  2. Created an account with Amazon Web Services (AWS)
  3. Created an OAuth consumer
  4. Set up AWS Lambda and integrated the access key into the application
  5. Wrote serverless functions for authentication
  6. Created and ran unit tests for all of the user stories
  7. Fixed and ran tests until I had 70-100% coverage
  8. Created and ran inegration tests for the application
  9. Added styling to the application
  10. Created acceptance tests for all features
  11. Created end-to-end tests for all features
  12. Created alerts using an object oriented approach
  13. Enabled service worker, and added icons to the application
  14. Implemented a Scatterplot, and a Pie Chart for data visualization
  15. Added a grid display for charts
